# Pacemaker

Resource planning for engineering teams that have outgrown spreadsheets.

Pacemaker replaces the planning spreadsheet with a computable model. Change an input — a priority, an allocation, a PTO block — and every downstream delivery date recalculates automatically.

## Who it's for

Engineering managers, TPMs, and PMs at organizations with 30–150 engineers running quarterly or half-year planning cycles. If you have 3+ skill types, cross-pod loans, and a spreadsheet that breaks by week 3, this is the tool you've been rebuilding every cycle.

## What it does

- **Skill-aware scheduling** — not "who's free" but "who has iOS skills AND is free starting week 7"
- **Real capacity math** — allocation percentages, PTO blocks, capability factors, pod-sharing splits
- **The cut line** — a visible boundary between committed and stretch, based on math, not vibes
- **Priority-based scheduler** — stack-rank projects, run the scheduler, see where capacity runs out
- **Cross-pod staffing** — one person, one timeline, allocations that actually sum to 100%
- **Delay tracking** — dates, reasons, and downstream impact, not just a yellow cell
- **Weekly check-ins** — 15-minute Mondays instead of 3-hour status report assembly
